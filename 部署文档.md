# 项目部署文档

## 环境要求

- Node.js (推荐版本 22.x 或更高)
- npm (推荐版本 10.x 或更高)
- Nginx (推荐版本 1.24 或更高)

## 部署步骤

### 1. 安装依赖

```bash
# 安装前端依赖
npm install
```

### 2. 构建项目

```bash
# 构建生产环境代码
npm run build
```

构建完成后，所有静态资源将被生成到 `dist` 目录中。

### 3. 部署到Nginx

1. 将 `dist` 目录中的所有文件复制到Nginx的网站根目录（通常是 `/usr/share/nginx/html` 或自定义目录）

2. 配置Nginx服务器，创建或修改配置文件（通常在 `/etc/nginx/conf.d/default.conf` 或自定义配置文件）：

```nginx
server {
    listen 8083;
    server_name your_domain.com;  # 替换为您的域名

    root /path/to/your/dist;  # 替换为实际的dist目录路径
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存配置
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 30d;
        add_header Cache-Control "public, no-transform";
    }

    # 禁止访问 . 开头的隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

3. 检查Nginx配置是否正确：
```bash
nginx -t
```

4. 重启Nginx服务：
```bash
systemctl restart nginx
```


## 常见问题

### 构建失败
- 检查Node.js版本是否符合要求
- 确保所有依赖都已正确安装
- 检查是否有权限问题

### 服务无法访问
- 检查Nginx服务是否正常运行
- 确认配置文件是否正确
- 检查防火墙配置
- 查看Nginx错误日志（通常在 `/var/log/nginx/error.log`）

## 维护建议

1. 定期更新依赖包
2. 监控服务器资源使用情况
3. 定期检查Nginx日志文件
4. 保持备份策略的执行
